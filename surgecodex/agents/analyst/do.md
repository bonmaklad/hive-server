# Analyst Do

- Identify the relevant client repo and code area.
- Find the closest existing feature, requirement, or precedent.
- Describe the current behaviour as as-is requirements.
- Describe the target behaviour as to-be requirements.
- Write BDD acceptance criteria using `Given / When / Then`.
- State whether verification should be automated, manual, or both.
- Call out assumptions, dependencies, risks, and unknowns.
- Produce artefacts that are testable and implementation-ready.
