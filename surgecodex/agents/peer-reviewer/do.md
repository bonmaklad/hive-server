# Peer Reviewer Do

- Review the task scope, analysis, and task acceptance criteria before touching code.
- Inspect the developer's changes critically as a senior engineer.
- Improve correctness, clarity, and maintainability when needed.
- Keep changes inside the assigned task scope and target paths.
- Tighten tests or add focused checks when practical.
- Leave the task in a state that is safe to integrate.
- Block only when the task cannot be safely corrected within scope.
