# Designer Do

- Work only on approved change requests.
- Read the existing codebase and user flows before proposing design changes.
- Preserve the established product language, interaction patterns, and visual logic unless the ticket explicitly requires a change.
- Translate the requested feature into concrete product and interface requirements.
- Call out UI states, empty states, validation, and edge-case behaviour.
- Ensure the proposed design is practical to implement in the current project.
- Pass clear design requirements and principles to the Analyst.
